# 开发工作流

> 此文件扩展 [common/git-workflow.md](./git-workflow.md) 添加 git 操作之前的完整功能开发流程。

功能实现工作流描述开发管道：研究、规划、TDD、代码审查，然后提交到 git。

## 功能实现工作流

0. **研究与复用** _(任何新实现之前强制)_
   - **首先进行 GitHub 代码搜索：** 在编写任何新内容之前运行 `gh search repos` 和 `gh search code` 查找现有实现、模板和模式。
   - **Exa MCP 用于研究：** 在规划阶段使用 `exa-web-search` MCP 进行更广泛的研究、数据摄取和发现现有技术。
   - **检查包注册表：** 在编写工具代码之前搜索 npm、PyPI、crates.io 和其他注册表。优先使用经过实战检验的库而非手写的解决方案。
   - **搜索可适配的实现：** 寻找解决 80%+ 问题并可以 fork、移植或包装的开源项目。
   - 在满足需求时，优先采用或移植经过验证的方法而非编写全新代码。

1. **先规划**
   - 使用 **planner** 代理创建实现计划
   - 在编码前生成规划文档：PRD、架构、system_design、tech_doc、task_list
   - 识别依赖和风险
   - 分解为阶段

2. **TDD 方法**
   - 使用 **tdd-guide** 代理
   - 先写测试（红）
   - 实现以通过测试（绿）
   - 重构（改进）
   - 验证 80%+ 覆盖率

3. **代码审查**
   - 编写代码后立即使用 **code-reviewer** 代理
   - 解决关键和高优先级问题
   - 在可能时修复中优先级问题

4. **提交与推送**
   - 详细的提交消息
   - 遵循 conventional commits 格式
   - 有关提交消息格式和 PR 流程，参见 [git-workflow.md](./git-workflow.md)

# /learn - 提取可重用模式

分析当前会话并提取值得保存为技能的任何模式。

## 触发器

在您解决非平凡问题的任何时候，在会话期间运行 `/learn`。

## 提取内容

查找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这对于类似错误可重用吗？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方法**
   - 库特性
   - API 限制
   - 版本特定修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建技能文件：

```markdown
# [描述性模式名称]

**提取：** [日期]
**上下文：** [此模式适用的简要描述]

## 问题
[这解决什么问题 - 具体说明]

## 解决方案
[模式/技术/变通方法]

## 示例
[代码示例（如适用）]

## 何时使用
[触发条件 - 什么应该激活此技能]
```

## 流程

1. 审查会话以查找可提取的模式
2. 识别最有价值/可重复使用的见解
3. 起草技能文件
4. 在保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注释

- 不要提取简单的修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定的 API 中断等）
- 专注于能在未来会话中节省时间的模式
- 保持技能聚焦 - 每个技能一个模式

# 前端幻灯片

创建零依赖、富有动画效果的HTML演示文稿，可完全在浏览器中运行。

灵感来自zarazhangrui展示的视觉探索方法（致谢：@zarazhangrui）。

## 激活时机

- 创建演讲文稿、推销文稿、研讨会文稿或内部演示文稿
- 将`.ppt`或`.pptx`幻灯片转换为HTML演示文稿
- 改进现有HTML演示文稿的布局、动画或排版
- 帮助尚不了解设计偏好的用户探索演示文稿风格

## 不可协商的原则

1. **零依赖**：默认使用单个自包含HTML文件，包含内联CSS和JS。
2. **强制适配视口**：每张幻灯片必须完全适配单个视口，不允许内部滚动。
3. **展示而非描述**：使用视觉预览而非抽象的风格问卷。
4. **独特设计**：避免使用通用的紫色渐变、白色背景上的Inter字体等模板化外观的演示文稿。
5. **生产级质量**：代码需带有注释、可访问、响应式且性能良好。

在生成之前，请阅读`STYLE_PRESETS.md`以获取视口安全的CSS基础、密度限制、预设目录和CSS注意事项。

## 工作流程

### 1. 检测模式

选择以下路径之一：
- **新建演示文稿**：用户已有主题、笔记或完整草稿
- **PPT转换**：用户已有`.ppt`或`.pptx`文件
- **增强**：用户已有HTML幻灯片并希望改进

### 2. 收集内容

仅询问必要信息：
- 用途：推销、教学、会议演讲、内部更新
- 长度：短（5-10页）、中（10-20页）、长（20+页）
- 内容状态：完整文案、粗略笔记、仅主题

如果用户已有内容，请在设置样式前让他们粘贴内容。

### 3. 探索风格

默认使用视觉探索方法。

如果用户已经知道所需的预设样式，请跳过预览直接使用。

否则：
1. 询问演示文稿应营造何种氛围：令人印象深刻、充满活力、专注、启发灵感。
2. 在`.ecc-design/slide-previews/`中生成**3个单页预览文件**。
3. 每个预览必须是自包含的，清晰展示排版/颜色/动画，且幻灯片内容保持在约100行以内。
4. 询问用户保留哪个预览或混合哪些元素。

在将情绪映射到样式时，请使用`STYLE_PRESETS.md`中的预设指南。

### 4. 构建演示文稿

输出以下之一：
- `presentation.html`
- `[presentation-name].html`

仅当演示文稿包含提取的或用户提供的图像时才使用`assets/`文件夹。

所需结构：
- 语义化幻灯片章节
- 来自`STYLE_PRESETS.md`的视口安全CSS基础
- 用于主题值的CSS自定义属性
- 用于键盘、滚轮和触摸导航的演示控制器类
- 用于揭示动画的Intersection Observer
- 减少动画支持

### 5. 强制适配视口

将此视为硬性要求。

规则：
- 每个`.slide`必须使用`height: 100vh; height: 100dvh; overflow: hidden;`
- 所有类型和间距必须使用`clamp()`缩放
- 当内容无法适配时，拆分为多张幻灯片
- 绝不通过将文本缩小到可读大小以下来解决溢出问题
- 绝不允许幻灯片内部出现滚动条

使用`STYLE_PRESETS.md`中的密度限制和强制CSS块。

### 6. 验证

在以下尺寸检查完成的演示文稿：
- 1920x1080
- 1280x720
- 768x1024
- 375x667
- 667x375

如果浏览器自动化可用，请使用它来验证没有幻灯片溢出且键盘导航正常工作。

### 7. 交付

在交接时：
- 删除临时预览文件，除非用户希望保留
- 当有用时，使用适合平台的打开器打开演示文稿
- 总结文件路径、使用的预设、幻灯片数量和简单的主题自定义要点

为当前操作系统使用正确的打开器：
- macOS：`open file.html`
- Linux：`xdg-open file.html`
- Windows：`start "" file.html`

## PPT / PPTX转换

对于PowerPoint转换：
1. 优先使用`python3`和`python-pptx`来提取文本、图像和笔记。
2. 如果`python-pptx`不可用，请询问是安装它还是回退到手动/基于导出的工作流程。
3. 保留幻灯片顺序、演讲者备注和提取的资源。
4. 提取后，运行与新建演示文稿相同的样式选择工作流程。

保持转换跨平台。当Python可以完成任务时，不要依赖仅适用于macOS的工具。

## 实现要求

### HTML / CSS

- 除非用户明确需要多文件项目，否则使用内联CSS和JS。
- 字体可来自Google Fonts或Fontshare。
- 优先使用大气背景、强烈的类型层次结构和清晰的视觉方向。
- 使用抽象形状、渐变、网格、噪声和几何图形而非插图。

### JavaScript

包含：
- 键盘导航
- 触摸/滑动导航
- 鼠标滚轮导航
- 进度指示器或幻灯片索引
- 进入时揭示动画触发器

### 可访问性

- 使用语义化结构（`main`、`section`、`nav`）
- 保持对比度可读
- 支持纯键盘导航
- 尊重`prefers-reduced-motion`

## 内容密度限制

除非用户明确要求更密集的幻灯片且可读性仍然保持，否则使用以下最大值：

| 幻灯片类型 | 限制 |
|------------|-------|
| 标题 | 1个标题 + 1个副标题 + 可选标语 |
| 内容 | 1个标题 + 4-6个项目符号或2个短段落 |
| 功能网格 | 最多6个卡片 |
| 代码 | 最多8-10行 |
| 引用 | 1个引用 + 出处 |
| 图像 | 1个受视口约束的图像 |

## 反模式

- 没有视觉标识的通用创业公司渐变
- 除非有意采用编辑风格，否则避免系统字体演示文稿
- 长项目符号墙
- 需要滚动的代码块
- 在短屏幕上会破坏的固定高度内容框
- 无效的否定CSS函数，如`-clamp(...)`

## 相关ECC技能

- `frontend-patterns`：用于演示文稿周围的组件和交互模式
- `liquid-glass-design`：当演示文稿有意借鉴Apple玻璃美学时
- `e2e-testing`：如果需要对最终演示文稿进行自动化浏览器验证

## 交付清单

- 演示文稿可从本地文件在浏览器中运行
- 每张幻灯片无需滚动即可适配视口
- 风格独特且有意
- 动画有意义而非嘈杂
- 尊重减少动画设置
- 在交接时解释文件路径和自定义要点

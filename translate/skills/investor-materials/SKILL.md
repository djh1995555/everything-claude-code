# 投资者材料

构建一致、可信且易于辩护的面向投资者的材料。

## 何时激活

- 创建或修订演示文稿
- 撰写投资者备忘录或单页介绍
- 构建财务模型、里程碑计划或资金使用表
- 回答加速器或孵化器申请问题
- 围绕单一事实来源对齐多个筹款文档

## 黄金法则

所有投资者材料必须相互一致。

在撰写前创建或确认单一事实来源：
- 增长指标
- 定价和收入假设
- 融资规模和工具
- 资金使用
- 团队简历和头衔
- 里程碑和时间表

如果出现冲突的数字，在起草前停下来解决它们。

## 核心工作流

1. 盘点规范事实
2. 识别缺失的假设
3. 选择资产类型
4. 使用明确逻辑起草资产
5. 对照事实来源交叉检查每个数字

## 资产指南

### 演示文稿
推荐流程：
1. 公司 + 切入点
2. 问题
3. 解决方案
4. 产品 / 演示
5. 市场
6. 商业模式
7. 增长
8. 团队
9. 竞争 / 差异化
10. 融资请求
11. 资金使用 / 里程碑
12. 附录

如果用户想要基于 Web 的演示文稿，将此技能与 `frontend-slides` 结合使用。

### 单页介绍 / 备忘录
- 用一句简洁的话说明公司做什么
- 展示为什么是现在
- 尽早包含增长和证明点
- 使融资请求精确
- 保持主张易于验证

### 财务模型
包括：
- 明确的假设
- 有用时的保守 / 基准 / 乐观情况
- 清晰的分层收入逻辑
- 与里程碑挂钩的支出
- 决策取决于假设时的敏感性分析

### 加速器申请
- 回答被问到的具体问题
- 优先考虑增长、洞察力和团队优势
- 避免夸大其词
- 保持内部指标与演示文稿和模型一致

## 应避免的危险信号

- 无法验证的主张
- 没有假设的模糊市场规模
- 不一致的团队角色或头衔
- 不能清晰求和的收入计算
- 在假设脆弱的地方夸大确定性

## 质量门

交付前：
- 每个数字都与当前事实来源匹配
- 资金使用和收入层求和正确
- 假设可见，不被隐藏
- 故事清晰，没有炒作语言
- 最终资产在合作伙伴会议上是可辩护的
